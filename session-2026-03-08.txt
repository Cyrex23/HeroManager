## 6. Seal Feature — Full Implementation (~17:52–18:24)
Implemented the complete seal system for heroes.

### Rules:
- Heroes earn +1 seal point every 4 levels
- Seal ranges from -10 to +10 (starts at 0)
- Each seal change costs 1 seal point (up or down by 1)
- A confirmation dialog is shown before changing seal

### Effect on combat:
- Seal controls the magic power damage RANGE
- At seal -10: MP damage rolls from 0% to 100% of MP (widest range)
- At seal 0: MP damage rolls from ~25% to 75%
- At seal +10: MP damage is fixed at exactly 50% of MP (no variance)
- Each seal level also affects Magic Proficiency %, Crit Chance %, Spell Activation %
  (defined in SEAL_STATS_TABLE, synced between frontend and backend)

### Backend:
- Hero entity: `seal` (int, default 0), `sealChanges` (int, default 0)
- sealPoints computed: floor(level/4) - sealChanges
- API endpoint: POST /api/heroes/{id}/seal with direction "up"/"down"
- PlayerService.getSealStats(seal) returns [magicProf%, critChance%, spellActivation%]
- BattleService uses seal to set the MP roll range in combat
- Files: Hero.java, PlayerService.java, PlayerController.java or HeroController.java,
         BattleService.java

### Sub Stats table updated to reflect seal:
- Magic Proficiency → from seal table
- Critical Chance → from seal table
- Spell Activation → from seal table

## 9. Seal UI — Damage Range Display (~18:53)
Added live damage range calculation inside the seal panel:
- Formula: dmgMin/Max = (PA/2) + (DEX * 0.33) + mpRange
- mpRange depends on seal level:
    rFactor = (10 - seal) / 20
    mpMin = MP * 0.5 * (1 - rFactor)
    mpMax = MP * 0.5 * (1 + rFactor)
- Shows breakdown: "PA [value] · DEX [value] · MP [min]–[max]"
- Labels show "PA" (not "PA/2") since PA value shown is already PA/2
